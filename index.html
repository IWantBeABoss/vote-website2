GitHub Pages 部署详细指南
提示： 按照以下步骤操作，您将能够在15分钟内将投票网站部署到GitHub Pages并获得一个公开网址！

1创建GitHub账户

如果您还没有GitHub账户，请先注册：

  1. 访问 GitHub官网
  2. 点击右上角的"Sign up"按钮
  3. 按照提示填写用户名、邮箱和密码
  4. 完成邮箱验证
注意： GitHub账户是完全免费的，您不需要支付任何费用。

2创建新仓库

  1. 登录GitHub后，点击右上角的"+"号，选择"New repository"
  2. 输入仓库名称，例如 vote-website
  3. 填写描述（可选），例如"一个简单的投票网站"
  4. 选择"Public"（公开）以便免费使用GitHub Pages
  5. 勾选"Add a README file"（可选，但建议）
  6. 点击"Create repository"按钮

3上传网站文件

将投票网站代码上传到您的GitHub仓库：

  1. 在仓库页面，点击"Add file" → "Upload files"
  2. 将本页面的HTML代码保存为index.html文件：
    • 右键点击页面，选择"另存为"
    • 将文件名改为index.html
    • 保存类型选择"网页，仅HTML"
  3. 将保存的index.html文件拖拽到GitHub的上传区域
  4. 在提交描述中填写"添加投票网站"
  5. 点击"Commit changes"提交

文件上传页面示意图

将文件拖到这里或点击上传

Commit changes

4启用GitHub Pages

  1. 在仓库页面，点击"Settings"选项卡
  2. 在左侧菜单中找到"Pages"（页面）
  3. 在"Source"部分，选择"Deploy from a branch"
  4. 选择分支（通常是main或master）
  5. 选择文件夹为/(root)
  6. 点击"Save"保存
  7. 等待几分钟，页面会显示一个网址，格式为：https://您的用户名.github.io/仓库名

5访问您的网站

完成以上步骤后：

  1. 打开浏览器，访问您的GitHub Pages网址
  2. 网址格式为：https://您的用户名.github.io/vote-website
  3. 如果看到投票网站，恭喜您部署成功！
恭喜！ 您的投票网站现在已经上线，可以通过互联网访问了！

6自定义域名（可选）

如果您有自己的域名，可以将其指向GitHub Pages：

  1. 在仓库的"Settings" → "Pages"中
  2. 找到"Custom domain"部分
  3. 输入您的域名，例如vote.yourdomain.com
  4. 点击"Save"
  5. 在您的域名注册商处添加CNAME记录，指向您的用户名.github.io
注意： 自定义域名需要您已经拥有一个域名，并且知道如何配置DNS记录。

常见问题解答

请检查：1) 文件名是否为index.html；2) 文件是否在仓库的根目录；3) GitHub Pages设置是否正确。

GitHub Pages需要几分钟时间构建和部署更改。如果超过10分钟仍无变化，请检查您的提交是否成功。
